# ② Link Layer (Sicherungsschicht)

> [!abstract] Auf einen Blick
> Der **Link Layer** (Ebene 2) sorgt für die fehlerarme Kommunikation zwischen **physisch verbundenen** Stationen. Er hat zwei Teilaufgaben: **LLC** (Logical Link Control — Datensicherung) und **MAC** (Media Access Control — Regelung des Medienzugangs).

## Notes in diesem Kapitel
1. [[2 Link Layer/LLC – Sicherung\|LLC – Sicherung]] — Frames, Fehlerbehandlung (ARQ/FEC), Flusskontrolle
2. [[2 Link Layer/MAC – Medienzugang\|MAC – Medienzugang]] — Multiplexing, Token, Random Access
3. [[2 Link Layer/Vermittlungsarten\|Vermittlungsarten]] — Circuit Switching vs. Packet Switching

## Roter Faden
Daten werden in [[2 Link Layer/LLC – Sicherung\|Frames]] gesendet, deren Korrektheit gesichert wird. Teilen sich mehrere Stationen ein Medium, regelt die [[2 Link Layer/MAC – Medienzugang\|MAC-Schicht]] den Zugriff. Auf der nächsthöheren Ebene stellt sich die Frage, **wie** Daten durchs Netz fließen: als reservierter Strom oder als einzelne Pakete ([[2 Link Layer/Vermittlungsarten\|Vermittlungsarten]]). Konkrete Umsetzungen folgen in [[3 Access Networks/3.0 Access Networks (Übersicht)\|Access Networks]].

> [!info] Dicke der Teilschichten
> Im **Access**-Netz ist primär **MAC** relevant (viele Rechner, ein Medium); im **Backbone** primär **LLC** (Point-to-Point, höhere Fehlerraten).
